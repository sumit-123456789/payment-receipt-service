# Payment Receipt Service

A Kotlin Spring Boot backend that provides a privacy-friendly digital receipt service for contactless payment demonstrations.

The service receives receipt data from a Point-of-Sale (POS) or Android application, stores it in memory (v1), and generates a unique URL that customers can access by scanning a QR code printed or displayed after checkout.

The goal is to demonstrate a modern alternative to traditional email receipts without requiring customers to share personal information during checkout.

---

## Features

### Current (v1)

* Kotlin + Spring Boot REST API
* In-memory receipt storage
* Generate unique receipt IDs
* Browser-friendly HTML receipt page
* QR code-based receipt retrieval
* Print / Save as PDF from browser
* Mobile-responsive receipt page

---

## Workflow

```text
Customer completes checkout
        │
        ▼
Android POS uploads receipt
        │
        ▼
POST /receipts
        │
        ▼
Receipt stored in memory
        │
        ▼
Receipt URL returned
        │
        ▼
QR Code generated by Android app
        │
        ▼
Customer scans QR code
        │
        ▼
Receipt opens in browser
        │
        ▼
Print / Save PDF / Share
```

---

## Why QR Receipts?

Traditional digital receipts usually require customers to provide an email address.

This service demonstrates a more privacy-friendly approach by allowing customers to retrieve their receipt using a QR code.

### Benefits

* No email address required
* Faster checkout
* Works on Android and iPhone
* No customer app required
* Easy receipt retrieval
* Browser-based experience
* Foundation for cloud-hosted receipts

## REST API

### Create Receipt

**POST**

```text
/receipts
```

Example request:

```json
{
  "storeName": "Art Int Bakery",
  "receiptText": "ART INT BAKERY\nTOTAL: $10.00",
  "totalCents": 1000
}
```

Example response:

```json
{
  "receiptId": "AB12CD34",
  "receiptUrl": "http://localhost:8080/r/AB12CD34"
}
```

---

### Get Receipt (JSON)

**GET**

```text
/receipts/{receiptId}
```

Returns the stored receipt as JSON.

---

### View Receipt (HTML)

**GET**

```text
/r/{receiptId}
```

Returns a mobile-friendly HTML receipt suitable for:

* Viewing in a browser
* Printing
* Saving as PDF
* Sharing via browser

## Testing

Create a receipt using curl:

```bash
curl -X POST http://localhost:8080/receipts \
-H "Content-Type: application/json" \
-d '{
  "storeName":"Art Int Bakery",
  "receiptText":"ART INT BAKERY\nTOTAL: $10.00",
  "totalCents":1000
}'
```

Open the returned `receiptUrl` in your browser to view the receipt.

## Current Limitations

This version intentionally keeps the implementation simple for demonstration purposes.

* In-memory storage only
* Receipts are lost when the application restarts
* No authentication
* No database persistence
* No PDF generation on the server
* No email delivery
